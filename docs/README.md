**Contents:**

- Comparative analyses

## Contributing Documentation

When adding documentation:

1. **Use clear, accessible language** (avoid unnecessary jargon)
2. **Include diagrams/visuals** where helpful
3. **Provide concrete examples** over abstract concepts
4. **Reference sources** for technical claims
5. **Keep it updated** (mark outdated content clearly)

See [CONTRIBUTING.md](../CONTRIBUTING.md) for contribution guidelines.

---

## Documentation Standards

- **Format:** Markdown (.md) for text, SVG for diagrams
- **Naming:** Use kebab-case (like-this.md)
- **Structure:** Start with overview, then details
- **Links:** Use relative paths for internal links
- **Images:** Store in `/docs/assets/images/`
